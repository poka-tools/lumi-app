## 1. 概要 / 目的

夜職（キャバクラ/クラブ・ガールズバー/コンカフェ・ラウンジ/スナック）で働く人が、
**出勤シフトと各種バックから当月の給料を概算し、現状の稼ぎを管理する**ためのアプリ。

- 「概算」が目的。1円単位の正確さより、**今月いくらになりそうか**をすぐ把握できることを優先する。
- バック額は店ごとに固定せず、**ユーザーが自分の店の条件を項目として登録**する設計。これにより A〜C のどの店種でも単一の仕組みで対応する。

### 進め方（A→B）
1. **試作フェーズ**: PWA（Webアプリ）として作り、自分のスマホで使用（ホーム画面に追加）。
2. **配布フェーズ**: 仕様が固まったら Flutter 等へ移植し、App Store / Google Play へ配布。
   - 移植コストを下げるため、**計算ロジックを純粋関数として UI から完全分離**しておく。
   - iOS 実機ビルドには Mac（またはクラウド Mac / EAS 相当）が必要。ストア申請操作はユーザー本人が行う。

### 対象 / 非対象
- 対象店種: A. キャバクラ/クラブ、B. ガールズバー/コンカフェ、C. ラウンジ/スナック
- 非対象: ホスト（売上歩合・スライド制など別体系のため v1 では扱わない）
- **目標機能は入れない**（モックアップの目標リング枠は「見込み合計」の強調表示に置換）

## 2. 画面構成（下タブ 5 つ）

| タブ | 役割 |
|---|---|
| **ホーム** | 当月の見込みダッシュボード（後述のモックアップ準拠） |
| **カレンダー** | 月表示。各日に出勤シフトと確定/未確定を表示。日タップで入力へ |
| **収入を記録 (＋)** | 中央タブ。出勤日の入力（開始/終了時刻・休憩・各バック項目の件数 or 売上） |
| **レポート** | 月別・項目別の集計とグラフ |
| **設定** | プロフィール、時給設定、バック項目の登録/編集、キャンペーンお知らせ設定、データ入出力 |

### ホーム画面の要素（モックアップ準拠）
- ヘッダー: マスコット（フクロウ）＋挨拶（「おはようございます、◯◯さん」）
- **当月の見込み合計**（確定前バッジ）＋**前月比**（差額・%）
  - 目標リングは入れない → このエリアは見込み合計を大きく強調表示
- サブ内訳: 時給（基本給） / インセンティブ（＝バック総額） / 総勤務時間
- ステータスチップ（横スクロール）: 出勤日数・時給換算・バック総額・本日見込み
- **キャンペーンお知らせバナー**（期間付き・表示のみ。設定で登録）
- 今月の収入サマリー: **収入の内訳ドーナツ**（時給 vs インセンティブ）＋**バック TOP3** ランキング
- クイックメニュー（収入を記録 / シフト管理 / 収入レポート / 日別収入 など）
- 直近のシフト・実績（横スクロール、日別の確定額・実働時間・未確定表示）

## 3. データモデル

端末内（IndexedDB）に保存。外部送信なし。

```
Profile
  name        : 表示名
  mascot      : マスコット種別（任意）
  hourlyWage  : 時給（円）
  storeName   : 店名（任意）

BackItem  （バック項目マスタ。ユーザーが自由に登録）
  id
  name        : 例) 同伴 / 本指名 / ドリンクバック / ボトルバック / チェキバック
  type        : "fixed"（固定円） | "rate"（歩合%）
  value       : type=fixed のとき単価(円) / type=rate のとき率(%)
  order       : 表示順

Shift  （出勤 1 回分の記録）
  id
  date        : 日付
  start, end  : 開始/終了時刻
  breakMin    : 休憩（分）
  workedHours : 実働時間（自動算出 = (end-start) - 休憩）
  confirmed   : 確定フラグ（true=確定 / false=未確定・予定）
  entries[]   : その日のバック実績
      backItemId
      count     : type=fixed の項目 → 件数
      sales     : type=rate の項目 → 対象売上(円)

Announcement  （キャンペーンお知らせ。表示のみ）
  id
  title       : 例) 5/31まで！ドリンクバックが+10%アップ中
  body        : 補足文（任意）
  startDate, endDate : 表示期間
```

## 4. 計算ロジック（純粋関数 `calc.js` に隔離）

入力（Profile / BackItem[] / Shift など）を受け取り、数値を返すだけの純粋関数群。
UI・DB に依存させない（＝ Flutter 移植時にそのまま仕様を移せる）。

- **時給分** = `hourlyWage × workedHours`
- **固定円バック** = `value × count`
- **歩合%バック** = `sales × (value / 100)`
- **その日の概算（日収）** = 時給分 ＋ その日の全バック合計
- **当月の見込み合計** = 当月の全 Shift の日収合計
- **前月比** = 当月見込み − 前月見込み（前月データがあるときのみ。差額と % を返す）
- **時給換算** = 当月見込み合計 ÷ 当月総実働時間
- **収入の内訳** = 時給分合計 vs バック総額（ドーナツ用に比率も）
- **バック TOP3** = BackItem 別の合計額を降順に並べ、上位 3 件と各構成比
- **本日見込み** = 今日の Shift の日収（未確定でも計算）
- **確定/未確定**: confirmed=false の Shift も見込みには含めるが「確定前」バッジで区別

## 5. 技術スタック / 保存

- **バニラ JS の PWA**（HTML / CSS / JS、フレームワーク無し）
  - `manifest.json` ＋ Service Worker でオフライン動作・ホーム画面追加に対応
- **データ保存: 端末内 IndexedDB**（外部サーバ無し → プライバシー上も安全）
- **グラフ**: ドーナツ/リングは Canvas 自前描画 or 軽量実装（依存を増やさない）
- **デザイン**: モックアップ準拠（ピンク基調・角丸カード・フクロウマスコット・余白広め）
- **データのエクスポート / インポート（JSON）** を設定に用意
  - 機種変更時の引き継ぎ、および Flutter 移植時のデータ移行に使う

## 6. 非機能・方針

- **オフライン前提**: ネット接続不要で完結（夜職の勤務中でも使える）。
- **プライバシー**: 収入データは端末内のみ。外部送信・アカウント登録なし。
- **テスト**: `calc.js` の各関数に単体テストを付け、概算ロジックの正しさを担保する。
- **移植容易性**: ロジックと UI を分離し、Flutter 移植時は `calc.js` の仕様を移すだけで済む形にする。
