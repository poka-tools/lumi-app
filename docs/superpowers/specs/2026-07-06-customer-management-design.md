# 顧客管理 & 来店予定機能 設計書

作成日: 2026-07-06
対象: Lumi（夜職の給与管理PWA / yashoku-salary）

## 目的

キャストが担当顧客を管理し、顧客ごとの来店予定を日付で登録してカレンダーに反映できるようにする。既存のシフト・給与計算ロジックには影響を与えず、独立した機能レイヤーとして追加する。

## スコープ

含む:
- 顧客リスト（追加・編集・削除・名前検索）
- 顧客ごとの来店予定（日付単位の登録・来店実績トグル）
- カレンダーへの来店予定表示
- ホームへの誕生日・近日来店予定の表示
- 設定を下タブから外し、右上⚙️ボタンへ移動

含まない（YAGNI）:
- ランク／タグ機能（顧客の格付け・絞り込み）※今回は実装しない
- 来店予定と売上・バック（実績金額）の紐づけ
- シフトの `指名あり`/`同伴あり` フラグの置き換え（現状のまま残す）
- 外部連携（LINE連携・通知プッシュ等）

## データモデル（IndexedDB: DB v2 → v3、既存データ保持）

`js/db.js` の `onupgradeneeded` に2ストアを追加。既存の汎用CRUD（`getAll`/`get`/`put`/`del`）をそのまま利用する。

### customers ストア（keyPath: id）
```
{
  id: string,            // uid()
  name: string,          // 氏名または源氏名（必須）
  contact: string,       // LINE ID / 電話番号など自由文字列
  birthday: string,      // "MM-DD"（年なし可）／空可
  favoriteBottle: string,// 好みのボトル・ドリンク
  memo: string,          // 自由メモ
  createdAt: number      // Date.now()
}
```

### visits ストア（keyPath: id）
```
{
  id: string,            // uid()
  customerId: string,    // customers.id への参照
  date: string,          // ISO "YYYY-MM-DD"
  note: string,          // 任意メモ（同伴・イベント等）
  done: boolean,         // 来店実績トグル（予定→来た✓）
  createdAt: number
}
```

visits は日付だけの軽量な予定レコード。既存の `shifts` と同じく、`getAll('visits')` をメモリ内で日付フィルタして使う（インデックスは張らない。データ量が小さいため既存パターンに合わせる）。

## UI構成

### 1. ナビゲーション変更（index.html / app.js）
- 下タブ: `🏠ホーム / 📅カレンダー / ＋ / 📊レポート / 👤顧客`
- `settings` はタブから削除するが**ルートとしては残す**（`navigate('settings')` で到達可能）
- 右上に固定の⚙️ボタンを追加し、タップで設定画面へ遷移（`position:fixed` 右上・全ビュー共通・スプラッシュ非表示後に表示）
- `app.js` の `renderers` に `customers: renderCustomers` を追加

### 2. 顧客タブ（js/ui/customers.js 新規）
- 一覧: 顧客カードに 名前 / 次回来店予定日（あれば） を表示
- 上部に名前検索ボックス（インクリメンタル絞り込み）
- 「＋顧客追加」ボタン → ボトムシート入力（既存 calendar.js のシート流儀に合わせる）
  - 入力項目: 名前 / 連絡先 / 誕生日(MM-DD) / 好みのボトル / メモ
- 顧客カードタップ → 詳細ビュー
  - 基本情報表示＋「編集」「削除」
  - 来店予定リスト（予定日・メモ・来た✓トグル・削除）。過去/未来でセクション分け
  - 「来店予定を追加」→ 日付＋メモ入力
- テキストは全て `esc()` でエスケープ

### 3. カレンダー連携（js/ui/calendar.js）
- 来店予定がある日のセルに 👤 バッジ表示（既存の 📌 Todoバッジと同様の仕組み）
- 日別ボトムシート上部に「この日の来店予定」セクション: 顧客名＋来た✓トグル。タップで当該顧客詳細へ遷移も可

### 4. ホーム連携（js/ui/home.js）
- 「本日の予定」カード付近に、以下を最大1カードで表示（既存 renderReminder と同じ枠組み）
  - 今月が誕生日の顧客（`birthday` の MM が当月）
  - 7日以内の来店予定
- 該当がなければカード非表示

### 5. 純粋関数・テスト
- 日付ロジック（当月誕生日判定、指定日/期間の来店予定抽出、次回来店予定の算出）を純粋関数として切り出す（`js/format.js` へ追加、または新規 `js/customers-logic.js`）
- `tests/` に `node --test` で単体テストを追加

## 検証方針
- `node --test` で純粋関数テスト全PASS
- SWキャッシュ回避のため別ポート（`python3 -m http.server 8123`）でローカル目視: 顧客追加→来店予定登録→カレンダー👤バッジ→ホーム表示 の一連を確認
- 既存データ（shifts/todos/backItems）が DB v3 マイグレーション後も保持されることを確認

## 後方互換・リスク
- DB v2→v3 は新ストア追加のみ（既存ストア変更なし）でデータ消失リスクなし
- SWキャッシュの新規JS追記漏れに注意（過去にexport不整合でスプラッシュ固着の事例あり）
